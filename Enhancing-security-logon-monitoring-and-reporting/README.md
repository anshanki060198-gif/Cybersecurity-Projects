
# Enhancing Security Logon Monitoring and Reporting

## Overview

This project focuses on enhancing security logon monitoring and reporting through the analysis of authentication logs and security events. The objective is to detect suspicious login activities, monitor user access, and improve security visibility within an organization.

## Project Objectives

* Monitor user logon and logoff activities.
* Detect failed login attempts and account lockouts.
* Identify potential brute-force attacks.
* Analyze authentication-related security events.
* Generate reports for security monitoring and auditing.

## Tools and Technologies

* Splunk
* Windows Event Viewer
* Windows Security Logs
* SIEM Concepts
* Log Analysis Techniques

## Methodology

### Log Collection

Security logs were collected from Windows systems to monitor authentication activities and user access events.

### Event Analysis

Authentication logs were analyzed to identify:

* Successful logins
* Failed login attempts
* Account lockouts
* Privileged account usage
* Suspicious authentication behavior

### Monitoring and Detection

Security monitoring techniques were applied to detect:

* Repeated failed logins
* Unauthorized access attempts
* Credential misuse
* Potential brute-force attacks

### Reporting

Reports were generated to provide visibility into:

* User authentication activity
* Security incidents
* Account lockouts
* Access trends and anomalies

## Outcomes

* Improved visibility into authentication events.
* Enhanced detection of suspicious login activity.
* Better security monitoring and reporting.
* Strengthened incident investigation capabilities.
